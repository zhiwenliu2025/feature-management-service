# Feature Management Service

*Design a Feature Management Service for an e-commerce platform. The system manages thousands of feature flags across >100 applications and services — covering web portals, backend APIs, and mobile clients. It needs to serve flag evaluations at high throughput with low latency while keeping resource usage within reasonable bounds as the feature catalog grows.*

**Your design should cover:**

* A caching strategy that remains efficient and cost-effective at this scale
* A client SDK that is easy to integrate and works consistently across different client types
* Full API design — management, evaluation, and any supporting APIs
* An observability strategy so the team can monitor health, debug issues, and understand system behavior
* An explainability model so that for any evaluated flag, you can answer: is it enabled? For whom? In which region? Associated with which release? etc.

**Submit your solution to a Github repository and share the link with us.**
